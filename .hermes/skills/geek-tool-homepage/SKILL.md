# Geek Tool Homepage Rules

## Overview

Frame Nest 的首页是一个极客工具站入口，不是产品说明书、路线图、讨论稿或架构文档。首页只展示用户当下可以使用的工具入口、必要分类和状态；内部设计思路、扩展规划、实现策略、讨论结论都应留在代码结构、注释或文档中，不能出现在用户可见页面里。

页面气质应当克制、直接、精美：用户一眼看到有哪些工具、当前状态、点击哪里进入。不要用长段解释说服用户，也不要把“为什么这样设计”写给用户看。

## When to Use

- 修改 `app/page.tsx`、`app/HomeHub.tsx`、`app/home.module.css` 或其它首页入口组件时。
- 新增或调整首页卡片、分类、状态、预览图、导航入口时。
- 写首页中文文案、卡片文案、空状态文案、按钮文案时。
- 用户指出首页“废话太多”“文案很虚”“把思考写进页面”时。
- 不适用于具体工具子页面的完整功能说明；子页面可以承载工具本身的必要操作说明。

## Non-Negotiable Rules

1. **不要把内部思考显示给用户**
   - 产品规划、扩展设想、架构意图、任务拆解、设计讨论，都不能渲染到首页。
   - 可以在代码中保留可扩展的数据结构，但用户可见文案必须只服务于当前入口。

2. **首页不是说明书**
   - 不解释“当前站点把哪些能力放在一个入口里”。
   - 不解释“新增页面时按分类扩展”。
   - 不解释“主页只负责入口、分类和状态”。
   - 不告诉用户“后续可以继续加入数据工具、生成器、地图页、3D 体验或临时实验”。

3. **卡片优先**
   - 每个功能入口是一个精美卡片。
   - 卡片应包含清晰标题、极短副标题/状态、视觉预览或图形暗示、明确的进入动作。
   - 不要让卡片承担长篇解释。

4. **当前首页只显示两个主要工具卡片**
   - 太阳系 / 宇宙此刻：链接到 `/cosmic-moment`。
   - 月相 / 海报实验室：链接到 `/poster-lab`。
   - 旅行地图、实验工具、扩展功能、未来能力不要作为首页可见卡片出现，除非用户明确要求恢复。

5. **文案必须功能型、短句化**
   - 好：`太阳系`、`查看行星位置`、`月相`、`生成月相海报`、`已上线`、`打开`。
   - 差：产品宣言、讨论稿、路线图说明、抽象愿景、诗句式占位、情绪化套话。

6. **保留扩展结构，但不暴露扩展文案**
   - 可以使用数组/配置来管理工具卡片，方便后续增加更多工具。
   - 可以保留分类字段用于筛选或布局。
   - 但不要显示“为更多工具预留结构”“扩展能力”“规划分类”等内部概念。

## Forbidden Visible Copy

以下文案或近义改写禁止出现在首页用户可见区域：

- `当前站点把宇宙时间、海报生成、旅行地图和实验工具放在一个入口里。新增页面时按分类扩展，用户先选任务，再进入具体工具。`
- `为更多工具预留结构`
- `主页只负责入口、分类和状态。具体能力留在子路由内实现，后续可以继续加入数据工具、生成器、地图页、3D 体验或临时实验。`
- `Expansion Model`
- `规划分类`
- `实验功能`
- `嵌套路由示例`
- 任何向用户解释“后续扩展”“站点结构”“产品规划”的长段文本。

## Recommended Homepage Shape

首页建议结构：

1. 顶部品牌与极少量导航。
2. Hero 区：短标题 + 一句功能型说明；不要超过一行到两行。
3. 两张主要工具卡片：太阳系、月相。
4. 每张卡片自带精美预览：轨道、星体、月相盘、网格、微动效均可。
5. 必要状态标签：例如 `已上线`。
6. 移除大段指标区、规划区、扩展区、解释性分类区，除非它们只展示当前真实可用入口且没有废话。

## Copy Examples

可用：

- `Frame Nest`
- `精密工具入口`
- `选择一个工具开始。`
- `太阳系`
- `查看行星位置`
- `月相`
- `生成月相海报`
- `已上线`
- `打开`

慎用：

- `功能入口中心`：可以用，但比 `工具` 或 `Frame Nest` 更重。
- `分类`：只有当页面确实提供分类筛选时才显示。

禁止：

- `当前站点把...放在一个入口里`
- `后续可以继续加入...`
- `为更多工具预留结构`
- `用户先选任务，再进入具体工具`

## Implementation Guidance

- 优先修改首页组件和样式，不要影响子路由功能。
- 删除未使用的图标、数组、CSS 类，避免保留死代码。
- 如果保留配置数组，命名应贴近当前工具，例如 `tools`，不要命名为 `expansion` 或 `plannedCategories`。
- 首页只渲染当前可用的两个工具。其它已有路由不删除，只是不在首页展示。
- 可用 CSS 制作卡片预览，不必引入重型资源。
- 响应式必须保持：桌面两列精美卡片，移动端单列。

## Verification Checklist

- [ ] 首页用户可见区域只显示太阳系/宇宙此刻和月相/海报实验室两个主要工具入口。
- [ ] 页面没有“当前站点把...”“为更多工具预留结构”“主页只负责...”“Expansion Model”“规划分类”等禁用文案。
- [ ] 没有旅行地图、港澳攻略、实验工具作为首页卡片或规划展示。
- [ ] 没有长段解释性产品文案。
- [ ] 卡片具备视觉预览，不只是纯文本列表。
- [ ] `npm run check` 通过。
- [ ] `npm run build` 通过，或若环境限制失败，记录明确失败原因。
- [ ] 通过浏览器截图/视觉检查确认首页干净、精美、文案克制。
