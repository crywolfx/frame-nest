# Geek Moon Poster Design Skill for Codex

Use this repo-local skill whenever implementing the new dedicated image-poster generator page, especially the Moon Phase template.

## Product Goal

Build a dedicated poster-generation page that feels like the existing `/cosmic-moment` solar-system workstation, but focuses on fast, high-quality poster/image export. The Moon Phase template is the first template and must support ~30 phase states across the lunar cycle.

## Visual Direction

- Style: geek / aerospace workstation / dark neon / glass panels / precision instrument UI.
- Stay close to the current solar-system page visual language: dark cosmic background, technical controls, subtle neon accents, cinematic but usable.
- Avoid decorative nonsense and fake poetic copy. No random poem-like lines, no vague motivational text. Use direct functional Chinese labels only.
- UI should feel premium and production-oriented: template picker, phase selector, preview, canvas/poster controls, batch queue, export status.
- Use restrained motion: panel entrance, hover glow, phase thumbnail transitions, preview refresh transitions, batch progress. Do not create distracting animation loops.

## Responsive UX

- Desktop: editor/workbench layout with large preview, left/right controls, batch panel.
- H5/mobile: stacked layout, sticky primary actions, large tap targets (>=48px), controls grouped into collapsible cards if needed.
- Use fluid/hybrid layout, flexible images, CSS media queries/container-aware behavior.
- No horizontal scrolling on mobile.

## Moon Phase Asset Requirements

- Built-in phase images: around 30 states, square 1:1, ultra-high-resolution source, realistic style.
- If generated by image model, each file must be audited one-by-one before final acceptance.
- Each image must correspond to its intended phase:
  - New moon: nearly invisible dark disk / no lit crescent.
  - Waxing crescent: right-side thin lit crescent.
  - First quarter: right half lit.
  - Waxing gibbous: right side mostly lit, left side shadow.
  - Full moon: full disk illuminated.
  - Waning gibbous: left side mostly lit, right side shadow.
  - Last/third quarter: left half lit.
  - Waning crescent: left-side thin lit crescent.
- Keep orientation consistent across the set. Use a documented convention: Northern Hemisphere visual convention, right side grows before full moon, left side shrinks after full moon.
- Do not accept images with wrong orientation, missing disk, cartoon style, non-square framing, obvious artifacts, text baked into the moon, or inconsistent lighting.
- Final files should be optimized for web but preserve enough resolution for poster export.

## Poster Editor Requirements

- Same level as existing cosmic poster tooling:
  - Text content
  - Font family
  - Font size
  - Color
  - Text x/y position
  - Alignment
  - Aspect ratio / output size
  - Metadata toggle
  - Live preview auto-refresh with debounce
  - Export PNG
  - Batch generation
- Preview and export must share rendering logic so position and styling match.
- Metadata must be factual and concise: phase name, lunar day/index, date if applicable. No view labels, no poem text.

## Batch Requirements

- Support rows like: `date | phase/template | text` or a clear documented variant.
- Batch must render each row using its specified date/phase/template, not stale current UI state.
- Show per-row status and summary. Invalid rows should be reported clearly.

## Animation Requirements

- Use Framer Motion / CSS transitions consistent with existing project if already installed.
- Animate state changes, not core drawing loops.
- Respect performance: no expensive canvas generation on every frame; debounce live preview.

## Engineering Constraints

- Keep user-visible times in Beijing time if date/time is shown.
- Do not introduce ISO/UTC strings into user-facing UI.
- Avoid committing Next generated noise. If `next-env.d.ts` is a tracked generated file causing repeated dev/build diffs, handle the repository policy explicitly in the plan (e.g. normalize, untrack with ignore if appropriate, or explain why not). Do not keep repeatedly warning the user.
- Reuse existing cosmic poster utilities where appropriate, but avoid over-coupling the new page to the 3D solar-system scene.
- Add clean route naming, e.g. `/poster-lab` or `/moon-poster`, after inspecting existing app structure.

## Acceptance Checklist

- Dedicated page exists and is navigable.
- Moon Phase template includes about 30 phase states and maps date/cycle to phase correctly.
- Built-in phase images are square, realistic, high-res enough, and individually audited.
- PC and mobile layouts are usable.
- Text styling and placement controls work.
- Live preview auto-refreshes.
- Preview can be enlarged if the page offers small embedded preview.
- Batch export switches to each row's date/phase/template.
- No poem-like filler copy.
- `npm run check` and `npm run build` pass.
